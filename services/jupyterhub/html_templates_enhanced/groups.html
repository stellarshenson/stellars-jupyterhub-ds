Enforce per-user compose project
Group each user's docker run containers under their own compose project so they show up together in docker compose ls / Docker Desktop. Off = ad-hoc containers belong to no compose project; the user's own docker compose projects are unaffected either way. Any group on → on.
Allow override of compose project
Respect a project name the user supplies via docker compose -p <name> instead of rewriting it. Off = every container is pinned to the per-user project regardless of what the user typed. Any group on → on.
Access hub network
Grant access to the hub's docker network so user containers can reach hub services and other containers by DNS. Off = the hub network is hidden and inaccessible. Any group on → on.
