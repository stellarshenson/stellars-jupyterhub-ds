Design language

Every token, pill, control and widget on one page — check the system in both themes.
Unlisted: reachable only by URL (/hub/design-language), not part of the product. The JupyterHub metaphors — pills, meters, bars — are custom and themed by the same --color-* tokens that carry the antd surface, so a theme flip restyles everything at once.
Colour tokens
The single source of truth. Every colour in the product resolves to a --color-* variable — never a literal hex in a component. -soft variants are ~10–16% tints for fills and backgrounds; -ring is the focus halo. Theming = overriding these vars on [data-theme]; components stay untouched.

Build notes
Tokens are CSS custom properties on the [data-theme] root; components read var(--color-x) only. A theme is a second [data-theme="light"] block overriding the same names — no per-component light/dark branches. -soft/-ring are pre-baked rgba / color-mix, not element opacity (opacity bleeds text + borders). Swatch = 44px div. Watchout: never hardcode a hex in markup — a literal won't flip with the theme.
Type, scale & normal text
Two faces only — a system sans for all UI, JetBrains Mono for data, IDs and timers (always with tabular-nums). Font sizes, radii and spacing must come from the fixed scales shown here; don't introduce off-scale values. Text colour carries meaning: neutral / link / success / warning / danger.

Build notes
Two families: --font-sans (system-ui) for UI, --font-mono (JetBrains Mono, loaded once in <helmet>) for data. Numbers/timers add font-variant-numeric:tabular-nums so digits don't jitter on tick. Sizes/radii/spacing come from --text-*/--radius-*/--space-* — pull, don't type px. Watchout: pills use 9999px, not 50%, to stay capsule at any width.
Named palette — dim / normal / bright
Each semantic colour has three steps, derived with color-mix against surface and text — so they track the theme automatically. Use dim for inactive / empty, normal for the resting state, bright for hover / emphasis. Never hand-pick an intermediate shade.

Build notes
Steps are color-mix(in srgb, <token> X%, var(--color-surface)) for dim and … var(--color-text)) for bright — computed + theme-aware, never separate hex. Consume as var(--doh-green-dim) etc. dim = inactive/empty, normal = resting, bright = hover/emphasis. Watchout: the right-hand mix colour must be a real surface/text token — mixing toward transparent just lowers alpha, it won't dim.
Action buttons
Three intents — primary (accent fill), secondary (outline), danger — at two sizes: 32px on pages, 24px in list rows. At most one primary per view; reserve danger for destructive, irreversible actions. Icon weight follows the standalone-vs-list rule below.

Icons: wireframe by default, filled on demand. Tones — primary (cyan, go-to), secondary (gray, neutral), dangerous (red, destructive), warning (orange, caution). List icons stay wireframe and fill only for emphasis; button icons use the filled glyph when one exists.
Build notes
Flex row, gap for icon+label spacing (never margins). Heights are fixed: 32px page / 24px list, border-radius:var(--radius-md). Primary = accent bg+border; secondary = surface+border, accent on hover; danger = danger bg. Disabled = bg-subtle + cursor:not-allowed, no hover. Icon weight follows the standalone(fill)/list(stroke) rule. Watchout: set font-family:inherit on <button> or it renders in UA serif.
Pills, tags & chips
Status is always a coloured pill from the shared state palette — soft fill + matching dot, never colour-only text. Transient states (spawning / in-progress) pulse the dot. Tags are flat labels; chips cap overflow with +N; scope pills are clickable filters that light with an accent ring when selected.

Inline-flex, gap:6px, soft bg + matching text + 7px dot. Transient pulse = @keyframes doh-pulse on the dot only (opacity), nothing that shifts layout. Connection status — pulsing diode
A liveness indicator where severity is encoded in cadence, not just colour. Core dot and halo fade together (opacity only, no scale): Connected is subtle and slow (3.6s); Not responding is pronounced and 3× faster (1.2s). Click the topbar pill to toggle the two states.
Connected Not responding · 2m
Diode: the core dot and its halo blink together with a single opacity fade — no scaling, no layout shift. Connected fades gently and slowly (subtle, 3.6s); Not responding fades hard and 3× faster (1.2s), the same cadence as the in-progress pill. Honours prefers-reduced-motion with a steady halo.
Build notes
Two nested spans: solid core dot + absolutely-positioned halo (box-shadow blur). Both ride one shared keyframe, opacity only — doh-pulse-calm 3.6s alive, doh-pulse-alert 1.2s (3×) down. Severity = cadence; JS swaps it by reassigning style.animation. Watchout: animate opacity, not transform:scale (scaling nudges the pill baseline); freeze under prefers-reduced-motion.
Meters, bars & widgets
All bars share one shape — a rounded track with a rounded fill. Usage bars (high = bad) go accent → warning at ≥70% → danger at ≥90%. GPU bars are the exception: they keep the accent base and vary only the stripe hue (10 fixed cool hues, cycling) so colour identifies a device, not its load. TTL is reversed — a low timer is the normal end state, never an alarm.

TTL spec (read before building)
Fractionf = remaining / base where base is the lifetime the server was granted (e.g. 4h) — NOT a fixed max, NOT the wall-clock day. Bar width = f. It only ever shrinks; extending raises base and remaining together.
ReversedLow is the normal end of life, so colour goes the opposite way to a usage bar: blue while f ≥ 0.30, warning when f < 0.30, dim red when f < 0.10. Never full danger-red, never a flashing alarm — a server reaching its TTL is expected.
GlowThe clock glyph (not the bar) gets a soft glowing-blur pulse that scales with urgency: none while blue, faint at warn (2.4s), bright + fast at the end (1s). Pure filter: drop-shadow — no layout shift, honours reduced-motion.
LabelShort duration (4h 0m, 6m) in mono + tabular-nums, coloured to match the bar. Exact expiry timestamp on hover only.
Colour thresholds: usage bars (high = bad) are calm accent below 70%, full warning at ≥70%, full danger at ≥90%. GPU bars are the exception — every device keeps the calm accent-blue base; only the 45° stripe overlay carries one of 10 fixed identity colours spread across the whole wheel (cyan, orange, green, violet, a neutral gray, magenta, blue, amber, teal) and tuned on hue + value so each device stays distinct at a glance. Past 10 GPUs the colours cycle (marked ↻). The TTL bar is reversed — blue while ample, warning under 30% of base, dim red under 10% (a low timer is the normal end state, not an alarm). As time runs out the clock glyph picks up a soft glowing-blur pulse — barely there at warn, bright and fast at the end — so a near-expiry server catches the eye without a hard alert. Exact values live in a tooltip on hover, never as static text.
Build notes
Track = rounded bg-subtle span; fill = inner <i> with width:% + border-radius:9999px. Usage colour by threshold (accent <70, warning ≥70, danger ≥90). GPU = accent base + repeating-linear-gradient(45deg, hsl(…) 0 3px, transparent 3px 7px) stripe, hue/sat/light per device, cycles past 10. TTL is reversed (see its spec row). Boost glow = forward-only box-shadow (+x offset, −spread) on the fill, cropped by track overflow:hidden. Watchout: round every fill end for one shape language.
Notices, alerts, grants & feed
Inline messaging, all keyed to the state palette. Notices use a 3px coloured left edge + matching leading icon on a subtle fill. Banners are full soft-tint blocks. Feed-item icons sit centred in a category-tinted circle. The colour always names the state — green done, info in-flight, amber caution, red failure.

Form controls
Inputs are 32px tall, 6px radius, with a 3px accent-soft focus ring. Disabled controls drop to a bg-subtle fill + subtle text (mono for locked/identifier values). Switches, checkboxes and radios fill with accent when on. Segmented controls use a soft-fill active cell, not a hard border.

Conventions
Cross-cutting rules that keep screens coherent: one short relative-time format everywhere (exact stamp on hover); zebra rows at a 3% text tint; state always shown as a coloured pill (never colour-only); button and label text in Title Case, with acronyms (API, GPU) and units (7h) preserved.

Two weights, one rule
filled = standalone / primary · line = in-list

A standalone or primary icon (a status glyph, a card avatar, a play/stop control) carries the filled weight so it reads as a solid object. The same concept inside a dense list, table row or menu uses the line weight to stay quiet next to text. Tone (cyan / gray / red / orange) is independent of weight.

Build notes
24×24 viewBox, one <path>, 1.7 stroke, round caps/joins. Line = fill:none;stroke:currentColor; standalone = fill:currentColor;stroke:none. Colour inherits via currentColor from the parent. Open paths (chevrons, activity zig-zag) MUST stay fill:none — filling them is meaningless; standalone fills need closed shapes, with detail dots/lines re-added as a second stroked path over the solid body. Watchout: never fill an open polyline.
