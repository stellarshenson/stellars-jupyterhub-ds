116. **Task - Apply group config at spawn time with reserved-name protection** (v3.10.2): Pre-spawn hook now resolves all user's groups into one effective config (docker/gpu/env vars), replacing hardcoded group-name checks and global GPU wiring<br>
    **Result**: The Groups admin page has been a store of preferences only - this change makes it the actual control plane. New pure module `stellars_hub/group_resolver.py` with `resolve_group_config(user_group_names, all_group_configs, gpu_available, reserved_names, reserved_prefixes)` collapses the user's groups into `{env_vars, gpu_access, docker_access, docker_privileged, matched_groups, skipped_env_vars}`. Resolution rules: grants (GPU, Docker, privileged) OR-accumulate across all of the user's groups so once any group grants an access it cannot be revoked by another; env vars iterate groups in priority-descending order and take the first occurrence of each name so the higher-priority group wins on conflict; GPU is gated on a separate `gpu_available` flag (host hardware detected or forced) so a group's `gpu_access=true` has no effect if the host has no GPU; reserved env var names are silently stripped and returned in `skipped_env_vars`.
